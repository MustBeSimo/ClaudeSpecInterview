Instruction Guide

The SPEC Workflow

Transform vague ideas into detailed specifications through structured dialogue with Claude Code. Three simple steps.

Why It Matters

Building without a specification is building with hope. Assumptions stay hidden until rework reveals them. The SPEC workflow forces clarity before implementation.

The Process

Create

Write a minimal SPEC.md file. 2–5 paragraphs only. Include: core intent, user context, rough scope, key concerns. Don't pretend completeness.

Interview

Paste the prompt below into Claude Code. Claude reads your spec, asks hard questions. You answer truthfully. The dialogue refines the specification.

"read this @SPEC.md and interview me in detail using AskUserQuestionTool about: technical implementation, UI & UX, concerns, tradeoffs. Make sure questions aren't obvious. Be very in-depth. Continue until complete, then write the spec to the file."
Inspired by @trq212's insights on spec-driven development. Structured questioning reveals what you truly need to build.

Finalize

Claude writes the detailed specification to SPEC.md. Architecture crystallizes. You now have a blueprint before writing code.

The Outcome

Precision in intent precedes elegance in implementation. The specification isn't a constraint—it's a blueprint for inevitability.
